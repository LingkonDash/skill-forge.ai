# Database Schema — SkillForge AI

## Purpose of This Document

This document defines the exact MongoDB collection schemas. The agent implements models against this document only — field names, types, and relationships must match exactly. Do not add fields, collections, or indexes beyond what's defined here unless a later instruction explicitly asks for it.

Models live at `server/src/lib/db/models/*` per `folder-structure.md`, using Mongoose.

---

## 1. Users

```
_id            ObjectId
name           String, required
email          String, required, unique
image          String (URL), optional
provider       String, enum: ["email", "google", "demo"], required
passwordHash   String, required if provider === "email", never returned in API responses
createdAt      Date, default: now
```

**Indexes:** unique index on `email`.

**Notes:** Better Auth manages session/token storage separately per its own conventions (see `auth-rules.md`); this collection holds the application-facing user profile only.

---

## 2. Roadmaps

```
_id                  ObjectId
userId               ObjectId, ref: User, required
careerGoal           String, required
currentSkills        [String], required
studyHours           Number, required
difficulty           String, enum: ["beginner", "intermediate", "advanced"], required
learningStyle        String, enum: ["video", "reading", "practice", "mixed"], required
generatedRoadmap     Mixed (structured JSON — see shape below), required
estimatedDuration    String, required (e.g. "12 weeks")
isPublic             Boolean, default: false
tags                 [String], default: []
createdAt            Date, default: now
updatedAt            Date, default: now
```

**`generatedRoadmap` shape:**
```
{
  weeks: [
    {
      weekNumber: Number,
      title: String,
      topics: [String],
      resources: [String],
      project: String,
      estimatedTime: String
    }
  ],
  tips: [String]
}
```

**Indexes:** index on `userId`; index on `isPublic` + `careerGoal` (for explore/filter queries).

**Notes:** `isPublic` is what determines whether a roadmap appears in the community Explore library. `tags` and `isPublic` are set by the user, not generated by the AI.

---

## 3. Progress

```
_id                    ObjectId
roadmapId              ObjectId, ref: Roadmap, required, unique
completedTopics        [String], default: []
completionPercentage   Number, default: 0
lastUpdated            Date, default: now
```

**Indexes:** unique index on `roadmapId` (one progress record per roadmap).

**Notes:** `completionPercentage` is recalculated server-side whenever `completedTopics` changes — it is never set directly by the client. The calculation is `completedTopics.length / total topics in generatedRoadmap`.

---

## 4. Reviews

```
_id            ObjectId
roadmapId      ObjectId, ref: Roadmap, required
userId         ObjectId, ref: User, required
rating         Number, min: 1, max: 5, required
comment        String, required
date           Date, default: now
```

**Indexes:** index on `roadmapId`; compound unique index on `roadmapId` + `userId` (one review per user per roadmap).

---

## 5. Conversations

```
_id            ObjectId
userId         ObjectId, ref: User, required
roadmapId      ObjectId, ref: Roadmap, required
createdAt      Date, default: now
```

**Indexes:** compound index on `userId` + `roadmapId`.

**Notes:** A conversation is scoped to a single roadmap so the AI Career Coach always has a specific roadmap as context. A user may have multiple conversations across different roadmaps, but not multiple concurrent conversations for the same roadmap — the backend reuses the existing conversation if one exists.

---

## 6. Messages

```
_id                ObjectId
conversationId     ObjectId, ref: Conversation, required
role               String, enum: ["user", "assistant"], required
content            String, required
createdAt          Date, default: now
```

**Indexes:** index on `conversationId` (messages are always fetched scoped to and ordered within a conversation).

---

## 7. Relationships Summary

```
User ──1:N── Roadmap
Roadmap ──1:1── Progress
Roadmap ──1:N── Review
Roadmap ──1:N── Conversation
Conversation ──1:N── Message
User ──1:N── Review
User ──1:N── Conversation
```

---

## 8. Field Rules Worth Calling Out

- `completionPercentage` is always server-derived, never client-set.
- `generatedRoadmap` is written once by the AI Roadmap Agent and only overwritten wholesale on a full "regenerate" — never partially patched.
- `isPublic` and `tags` are the only two roadmap fields editable by the user after creation (aside from full regeneration).
- Passwords (`passwordHash`) must never be included in any API response — enforce this via the Mongoose schema's `toJSON` transform, not just by omission in controllers.
